# Analyst Notes

## Purpose
This document summarizes key observations and lessons learned while building this phishing email analysis portfolio project.

## General Observations
Different phishing emails use different pressure mechanisms. Some focus on account security, some on financial urgency, and some on authority or secrecy.

## Notes from Sample 001
The fake Microsoft password reset sample shows how attackers often imitate trusted services and use lookalike domains. A major lesson is that visible link text alone should never be trusted.

## Notes from Sample 002
The fake invoice sample shows how phishing can target users through financial pressure and attachment-based lures. The misleading double extension is an important warning sign.

## Notes from Sample 003
The BEC sample shows that phishing does not always require a link or attachment. Social engineering alone can be enough when urgency, secrecy, and executive impersonation are used together.

## Cross-Case Patterns
Common phishing themes observed across the current samples:
- urgency
- pressure
- impersonation
- attempts to reduce verification
- attempts to trigger fast action

## Analyst Mindset
A good phishing review should not focus on only one red flag. The strongest assessments usually come from combining multiple indicators such as sender anomalies, message intent, pressure tactics, and suspicious technical details.

## Safe Handling Lessons
Suspicious emails should be reviewed carefully and safely. Links should not be clicked directly, attachments should not be opened casually, and public portfolio samples should always be sanitized.

- documentation discipline
