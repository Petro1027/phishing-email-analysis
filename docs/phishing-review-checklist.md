# Phishing Email Review Checklist

## Purpose
This checklist is used to review suspicious emails in a safe, structured, and repeatable way.

## 1. Sender Review
- [ ] Check the display name
- [ ] Check the full sender email address
- [ ] Check for lookalike domains
- [ ] Check whether the reply-to address differs from the sender
- [ ] Check whether the sender matches the claimed organization

## 2. Subject Review
- [ ] Look for urgency
- [ ] Look for threats or deadlines
- [ ] Look for unusual wording
- [ ] Look for financial or account-related pressure

## 3. Body Content Review
- [ ] Check whether the email asks for credentials
- [ ] Check whether the email asks for payment
- [ ] Check whether the email asks for secrecy
- [ ] Check whether the email discourages verification
- [ ] Check for grammar or spelling anomalies
- [ ] Check for generic greetings

## 4. Link Review
- [ ] Check whether links are present
- [ ] Compare visible text with the real destination
- [ ] Look for suspicious domains
- [ ] Do not click suspicious links directly

## 5. Attachment Review
- [ ] Check whether an attachment is present
- [ ] Review the file extension carefully
- [ ] Look for misleading double extensions
- [ ] Treat unexpected attachments as suspicious

## 6. Technical Review
- [ ] Review headers if available
- [ ] Check SPF if available
- [ ] Check DKIM if available
- [ ] Check DMARC if available
- [ ] Note any anomalies in routing or authentication

## 7. Final Assessment
- [ ] Record the main red flags
- [ ] Assign a verdict
- [ ] Recommend an action
- [ ] Document the case in a report

## Common Red Flags
- Urgency
- Fear or pressure
- Credential requests
- Payment requests
- Secrecy
- Suspicious domains
- Link mismatch
- Unexpected attachments
- Executive impersonation
- Generic greetings

## Safe Handling Reminder
Never interact directly with suspicious links, attachments, or attacker-controlled infrastructure on the host machine.
